# motor_control.py 文档

## 概述

上位机（PC）通过串口与下位机（STM32）通信的电机控制终端程序。使用 curses 构建 TUI（终端用户界面），支持键盘控制、命令收发日志显示、自动重复发送等调试功能。

## 数据流路径

```
┌─────────────────────────────────────────────────────────────┐
│                        主线程                                │
│  ┌──────────┐  按键   ┌──────────────┐  十六进制字节流       │
│  │ stdscr   │ ──────→ │ send_command │ ──────────────────┐  │
│  │ .getch() │         │              │                   │  │
│  │  10ms    │         │  TX: ...日志  │                   │  │
│  │  轮询    │         └──────────────┘                   │  │
│  └──────────┘                                            ▼  │
│                                                       ┌────┴────┐
│  ┌─────────────────┐  每 10ms  ┌──────────────┐      │ serial  │
│  │  auto_repeat    │ ────────→ │ send_command │      │  .write │
│  │  (自动发送)      │          │ (重复上次)    │      └────────┤
│  └─────────────────┘          └──────────────┘               │
│                                                              │
│  ┌──────────────────────────────────────────────────┐        │
│  │            Log Window (curses子窗口)              │        │
│  │   TX: 14:30:01 AA 01 02 01 00 00 02 11 4B       │        │
│  │   RX: 14:30:01 CAN_TX OK ID=0x102                │        │
│  │   RX: 14:30:02 CAN_TX OK ID=0x102                │        │
│  └──────────────────────────────────────────────────┘        │
│                                                              │
│                        RX 后台线程                            │
│  ┌────────────┐  ┌──────────────┐  ┌──────────┐  ┌─────────┐│
│  │  serial    │  │  io.BytesIO  │  │ 按 \n   │  │ refresh ││
│  │  .read     │─→│  (行缓冲)    │─→│ 分割完整行│─→│ _log    ││
│  │            │  │ 不完整行存回   │  │          │  │         ││
│  └────────────┘  └──────────────┘  └──────────┘  └─────────┘│
└─────────────────────────────────────────────────────────────┘
                             │
                             ▼   串口线 (UART-TTL / USB转串口)
                    ┌──────────────────┐
                    │    STM32 网关     │
                    │  UART → CAN 桥   │
                    └──────────────────┘
                             │
                             ▼   CAN 总线
                    ┌──────────────────┐
                    │  电机驱动节点      │
                    │  ID: 0x123~0x126  │
                    └──────────────────┘
```

**数据流说明**:

1. **TX 路径**: 按键 → 查表得十六进制命令 → `serial.write()` → STM32 → CAN → 电机节点
2. **RX 路径**: 电机节点 → CAN → STM32 → `serial.read()` → `BytesIO` 行缓冲 → 按 `\n` 切分 → 日志窗口
3. **自动发送**: `auto_repeat_enabled=True` 时，主循环每 10ms 检查定时器，到间隔则重复 `send_command(last_command)`

## 函数说明

### 模块级函数

#### `select_port() → str`
- **用途**: 交互式选择串口
- **流程**: 扫描 `/dev/ttyACM*` + `/dev/ttyUSB*` → 列出可用端口 → 用户输入编号 → 返回路径
- **无端口时**: 返回默认 `/dev/ttyACM0`
- **位置**: L12-27

#### `refresh_log(log_lines, log_win, lock)`
- **用途**: 刷新 curses 日志子窗口，仅显示最后可见行
- **线程安全**: 内部 `with lock:` 保护 curses 操作
- **位置**: L34-45

#### `send_command(command_str, log_lines, log_win, lock)`
- **用途**: 解析十六进制命令并写入串口
- **参数**: `command_str` = 如 `"AA 01 23 01 00 00 02 11 4B"`
- **错误处理**: `ValueError`(hex 格式错) + `SerialException`(串口断开) 捕获并写入错误日志
- **串口保护**: `with serial_lock:` 包装 `ser.write()`
- **日志**: TX 日志在 `with lock:` 内追加，`refresh_log` 在锁外调用(避免死锁)
- **位置**: L47-62

#### `rx_reader(log_lines, log_win, lock, stop_event)`
- **用途**: 后台线程，持续读取串口接收数据
- **行缓冲**: 使用 `io.BytesIO` 累积数据，按 `\n` 切分，不完整行暂存
- **并发**: `serial_lock` 保护 `ser.in_waiting/read()`，`lock` 保护日志追加
- **断线处理**: `SerialException` 时记录错误日志并退出
- **间隔**: 无数据时 sleep(10ms)；有数据时处理完 sleep(10ms)
- **位置**: L64-96

### main 函数

#### `main(stdscr)`
- **用途**: 程序入口，curses wrapper 调用
- **模块**:
  1. **初始化** (L98-190): curses 设置、commands 字典、help_text、log 窗口、自动发送状态、RX 线程
  2. **主循环** (L197-226): 10ms 轮询按键，处理 q/r/命令键，自动发送
  3. **清理** (L227-230): 停止线程，关闭串口
- **位置**: L98-230

## 关键变量

| 变量 | 类型 | 作用域 | 说明 |
|------|------|--------|------|
| `ser` | `serial.Serial` | 全局 | 串口对象，所有收发操作的目标 |
| `serial_lock` | `threading.Lock` | 全局 | 串口读写互斥锁，防止 TX/RX 并发冲突 |
| `MAIN_LOOP_TIMEOUT_MS` | `int` | 模块级 | 主循环 `getch()` 超时(毫秒)，当前 10ms |
| `RX_POLL_INTERVAL` | `float` | 模块级 | RX 线程轮询间隔(秒)，当前 0.01=10ms |
| `AUTO_REPEAT_INTERVAL` | `float` | 模块级 | 自动发送间隔(秒)，当前 0.01=10ms |
| `commands` | `dict[int, str]` | main 局部 | 按键→十六进制命令映射表 |
| `log_lines` | `list[str]` | main 局部 | 日志行列表，每条带时间戳的 TX/RX/ERR 记录 |
| `log_win` | `curses window` | main 局部 | 日志子窗口对象 |
| `lock` | `threading.Lock` | main 局部 | 日志列表和 curses 操作的互斥锁 |
| `stop_event` | `threading.Event` | main 局部 | RX 线程停止信号 |
| `last_command` | `str \| None` | main 局部 | 最近发送的完整十六进制命令字符串 |
| `auto_repeat_enabled` | `bool` | main 局部 | 自动发送开关 |
| `last_send_time` | `float` | main 局部 | 上次发送的时间戳(`time.time()`) |

## 命令协议

### 帧格式

```
AA 01 <ID> 01 00 00 02 11 <DATA>
```

| 字节偏移 | 长度 | 说明 |
|----------|------|------|
| 0 | 1 | 帧头 `0xAA` |
| 1 | 1 | 源地址 `0x01`(PC→网关) |
| 2 | 1 | **目标 CAN ID** (电机节点地址) |
| 3 | 1 | 帧类型 |
| 4-5 | 2 | 保留 |
| 6 | 1 | DLC(数据长度) |
| 7 | 1 | 命令码 `0x11`(控制) |
| 8 | 1 | DATA: 控制值 |

### 控制值(DATA)

| 值 | 意义 |
|----|------|
| `0x4B`(75) | 正向(前进/左转) |
| `0x00` | 停止 |
| `0xB5`(181) | 逆向(后退/右转) |
| `0x32`(50) | 动力中速 |
| `0x3C`(60) | 动力前进 |
| `0xC4`(196) | 动力逆向 |

### 节点地址

| CAN ID | 说明 |
|--------|------|
| `0x101` | 全体停止(广播) |
| `0x102` | 全部转向电机 |
| `0x103` | 全部动力电机 |
| `0x123` | 左前轮-方向 |
| `0x124` | 左前轮-动力 |
| `0x125` | 右后轮-方向 |
| `0x126` | 右后轮-动力 |

### 调试查询命令

```
AA 01 <ID> 02 00 00 01 01
AA 02 <ID> 01 00 00 01 04  (调试日志开启)
AA 02 <ID> 01 00 00 01 05  (调试日志关闭)
```

### 依赖关系

- `main()` 调用: `send_command()`, `threading.Thread(rx_reader)`
- `send_command()` 引用: `ser`(全局), `serial_lock`(全局)
- `rx_reader()` 引用: `ser`(全局), `serial_lock`(全局)
- `send_command()` 与 `rx_reader()` 无直接调用关系，通过 `serial_lock` 互斥

## 线程模型

| 线程 | 职责 | 同步机制 | 间隔 |
|------|------|----------|------|
| **主线程** | curses 键盘轮询 + 命令发送 + 自动发送 | `lock`(日志互斥) + `serial_lock`(串口互斥) | 10ms 轮询 |
| **RX 线程** | 串口数据读取 + 行缓冲 + 日志刷新 | `lock` + `serial_lock` | 10ms sleep |

### 锁使用规范

- `serial_lock`: 保护 `ser.in_waiting`, `ser.read()`, `ser.write()` — 只应持锁微秒级
- `lock`: 保护 `log_lines.append()` 和 curses `addstr()` 操作 — `refresh_log()` 须在锁外调用
