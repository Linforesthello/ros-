# locationMode_8010t1.cpp 参数详细解读

## 概述

控制 Unitree GO-M8010-6 电机，以**缓变位置闭环**方式转到指定角度。程序从当前实际位置出发，以固定步长逐步逼近目标，避免瞬间大力矩冲击。

---

## 一、用户可调参数

### targetDeg —— 目标角度（度）

```
当前值: 0.0f
```

输出端（你看到的轴）要转到的位置，单位是**度**。`360.0f` = 转一整圈回到原位，`90.0f` = 转四分之一圈，`180.0f` = 转半圈。

> 内部自动用 `deg2rad()` 把度转成弧度再发给电机。

### stepSize —— 每步最大步进量（电机轴弧度）

```
当前值: 0.10f
```

每 200μs 循环最多往前走多少弧度（注意这里是**电机轴**的弧度，不是输出端）。值越小运动越慢越平滑，值越大越快但冲击越大。

换算参考（电机轴每步进量对应的输出端角度 ≈ 步进量 / |gearRatio| × 180/π）：
- `0.10f` ≈ 每步输出端约 0.90°
- `0.20f` ≈ 每步输出端约 1.81°
- `0.70f` ≈ 每步输出端约 6.34°

### gearRatio —— 减速比

```
值: -queryGearRatio(MotorType::GO_M8010_6)   // -6.33
```

电机内部有减速齿轮（6.33:1），电机轴转 6.33 圈，输出端才转 1 圈。这个值把"输出端的角度"换算成"电机轴的角度"。

**说明**：早期版本写为 `-6.33 * queryGearRatio(...)`，但 `queryGearRatio` 本身已经返回 `6.33`，额外再乘 6.33 会导致减速比被重复计算（实际变为 -40.07），使电机多转约 6.3 圈。去掉冗余系数后修正为 `-queryGearRatio(...)`。

**负号**的由来：M8010_6 的转子磁编码器安装方向与输出轴反向，负号用于在内环控制中补偿这个反向关系，使输出端正向角度对应正确的转子位置。

## 二、控制指令参数（MotorCmd 结构体）

每次 `serial.sendRecv(&cmd, &data)` 发给电机的指令：

### cmd.mode —— 电机运行模式

| 值 | 含义 |
|----|------|
| `FOC`（运行时） | 磁场定向控制，可同时控制位置、速度、力矩 |
| `0`（停止时） | 停止模式，电机无力矩输出 |

### cmd.id —— 电机 ID

```
值: 0
```

当总线上挂多台电机时区分设备。单台电机始终为 `0`。

### cmd.kp —— 位置刚度系数（位置闭环 P）

```
运行值: 0.15
```

电机偏离目标位置时的**回正力度**。可以想象成一根弹簧：

- **kp 越大** → 弹簧越硬 → 偏离目标时回正力越大 → 位置越准，但电机越"僵硬"，容易抖动
- **kp 越小** → 弹簧越软 → 手可以轻松推动 → 位置不准，但运动更柔顺

典型调节思路：
- 空载轻载：`0.05` ~ `0.20`
- 重载或需要高刚度：可增加到 `1.0` 以上

> **类比**：就像汽车的悬挂弹簧，太硬颠簸（抖动），太软晃悠（位置不准）。

### cmd.kd —— 阻尼系数（位置闭环 D）

```
运行值: 0.10
```

抵抗电机运动的**阻力**，像在油里搅动。它的作用是吸收动能，防止超调和震荡：

- **kd 太小** → 电机到达目标后会"冲过头"然后来回震荡
- **kd 太大** → 电机动作迟钝、发涩

kp 和 kd 需要配合调节：
- 增加 kp 后通常也要适当增加 kd，否则会震荡
- 经验比例：`kd ≈ kp × 0.5 ~ 1.0`

> **类比**：就像汽车的减震器，kd 控制"回弹速度"，没有它弹簧（kp）会弹个不停。

### cmd.q —— 目标位置（电机轴弧度）

```
值: cmdPos（动态变化，从当前位置逐步逼近 targetPos）
```

告诉电机"我想让你转到哪个角度"。单位是电机轴的弧度，程序内部已通过 `gearRatio` 将输出端目标度（`targetDeg`）转换为电机轴目标弧度（`targetPos`），再缓变赋值给 `q`。

### cmd.dq —— 目标速度（电机轴弧度/秒）

```
值: 0.0
```

期望的电机转速。当前程序只控制位置，不控制速度，设为 0。

### cmd.tau —— 目标力矩（Nm）

```
值: 0.0
```

期望的电机输出力矩。当前程序依靠位置闭环（kp/kd）自动产生力矩，不直接给定力矩值。

## 三、反馈参数（MotorData 结构体）

电机回复的数据，可以在日志中看到：

### data.q —— 当前实际位置（电机轴弧度）

电机此刻的真实角度。可以通过 `rad2deg(data.q / gearRatio)` 换算成输出端的度数。

### data.dq —— 当前实际速度（电机轴弧度/秒）

电机此刻的转速，单位是电机轴弧度/秒。如果这个值很大，说明电机在快速运动。

### data.temp —— 电机温度（℃）

电机当前温度。如果持续升高到 80℃+，需要停机冷却。

### data.merror —— 错误码

| 值 | 含义 |
|----|------|
| `0x00` | 正常 |
| 非零 | 出现错误（过流、过温、编码器异常等） |

## 四、撤力参数

### 撤力第 1 步（位置保持 → 缓慢松劲）

```
循环 100 次，每次 200μs，共约 20ms

kp: 0.15 → 0.00   （刚度逐渐消失，不再跟位置误差较劲）
kd: 0.10 → 0.20   （阻尼逐渐加大，吸收残余动能，不让电机乱晃）
```

### 撤力第 2 步（完全停止）

```
mode = 0    切到停止模式
q = 0        目标位置清零
dq = 0       目标速度清零
tau = 0      目标力矩清零
kp = 0       位置刚度归零
kd = 0       阻尼归零
```

## 五、程序流程速览

```
启动
  │
  ▼
读取当前实际位置 → cmdPos = data.q    ← 防止突然起步
  │
  ▼
主循环（每 200μs）
  ├─ 计算目标：cmdPos 逐步逼近 targetPos
  ├─ 发送指令：kp, kd, q=cmdPos
  └─ 打印反馈：gearRatio, q, temp, dq, merror
  │
 Ctrl+C
  │
  ▼
撤力第 1 步：kp 渐降 → 0，kd 渐升 → 0.2    ← 松劲吸能
  │
  ▼
撤力第 2 步：mode=0，全部清零               ← 完全停机
```
