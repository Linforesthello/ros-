# 远距离检测改进 — 历程与总结

## 核心矛盾

```
HSV掩码像素不足  ←→  任何检测算法都需要足够像素输入
```

经过多轮迭代，所有算法侧的改动（密度网格、轮廓质心、ROI裁剪、HSV放宽）都没能实质突破 ~2m 检测极限。**根本瓶颈不在检测算法，而在 HSV 分割阶段就已经丢失了目标。**

---

## 参数改动记录（gpt_Com_t1.py）

| 改动 | 旧值 → 新值 | 效果 |
|------|:-----------:|:----:|
| `grid_size` | 16 → **8** | 量化 ±16px → ±8px，检测边界从 ~1.65m → ~1.8m |
| `density_min_pixels` | 10 → **5** | 配套 grid_size 减半，无单独可测效果 |
| `radius_tolerance` | 0.7 → **0.8** | 放宽半径验证，无单独可测效果 |
| `_peak_center` 峰值门槛 | 5 → **2** | 允许单格仅 2 像素即可成峰，对空检测边界无实质突破 |
| 轮廓回退（dilate + findContours） | 无 → 添加 | 密度失败时回退，仍因掩码像素不足无效 |

### 各改动之间的交互

- grid_size 减半 + density_min_pixels 减半 → 格子数 x4，允许更稀疏的像素分布 → 检测边界 +0.15m
- `_peak_center` 门槛 5→2 → 单格峰值更易通过 → 但总像素数门槛（line 147）限制了最终效果
- 轮廓回退 → 依赖掩码连通域面积 ≥10px² → 2m 外掩码碎片化严重，dilate 也无法形成足够大的连通域
- 半径验证容差 0.7→0.8 → 不是瓶颈，排查 log 确认被拒的都是"无有效深度"或"先验门限拒绝"

---

## 曾被尝试但已撤销的方向

### 1. 全图轮廓替代密度网格（撤销）

用 `findContours + moments` 完全替代密度网格，不做距离判断。  
**结果**：近处（<1m）黄蓝色块分离，最大轮廓只能抓到单色半边，质心偏置严重。

### 2. ROI 裁剪 + 宽松 HSV（撤销）

在先验位置裁 160×160 窗口，用更低的 S/V 阈值重新分割。  
**结果**：宽松 HSV 在 ROI 内也捕获了大量背景噪声，虚警爆炸。

### 3. 去掉 `draw_radius > 5` 闸门（撤销）

让所有检测不经半径预检查直接进入后续验证链。  
**结果**：微小噪点进入验证链，深度查询偶尔命中有效值，导致误发布无用坐标。

---

## 关键结论

```
远距离检测失效的根本原因：
  0.5x 缩放后，2m 处球体直径 ≈ 38px → 黄色/蓝色各占约一半
  → 单色有效像素 ~200-300px → HSV 阈值过滤后只剩 ~20-50px
  → 密度图分散在 3-4 个 8×8 格子中 → 不足以形成可靠峰值
  → 轮廓回退也因碎片化面积 < 10px² 而被过滤
```

### 可能的真正解决路径

需要突破的瓶颈是 **输入信号量**，而非算法：

| 方向 | 原理 | 代价 |
|------|------|------|
| **取消 0.5x 缩放** | 全分辨率处理，远距离像素量 x4 | 计算量 x4，回调耗时增加 |
| **原图 ROI + 不缩放** | 先验位置裁原图区域，全分辨率检测 | 需要 ROI 裁剪 + 独立处理路径 |
| **降低黄色 S 阈值** | 黄色 S=100 比蓝色 S=50 严格一倍，降到 60 | 可能引入轻微黄色噪声 |
| **换更高分辨率相机** | 从 640×480 → 1280×720 | 硬件成本 |

**推荐优先级**：先试"ROI 内不缩放"（纯软件改动、可控），再试"降低黄色 S 阈值"（1 行参数改动）。
