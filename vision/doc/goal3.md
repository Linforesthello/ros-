# Goal 3：中远距离（1~4m）识别优化

> 修改文件：`good/gpt_CommandLine.py`（仅改 3 处参数）

---

## 一、问题根因定量分析

相机缩放后 fx_scaled ≈ 300（原始 fx≈600，img_scale=0.5），球半径 0.12m。  
各距离下像素半径 `r = fx × R / Z`，单色掩码面积按 60% 覆盖率 × 半球估算：

| 距离 | 像素半径 | 理想面积 | 单色连通域估算 |
|------|---------|---------|-------------|
| 1 m  | 36 px   | 4072 px² | ~1222 px²  |
| 2 m  | 18 px   | 1018 px² | ~305 px²   |
| 3 m  | 12 px   | 452 px²  | ~136 px²   |
| 4 m  |  9 px   | 254 px²  |  ~76 px²   |

### 瓶颈一（先验采集阶段）—— `area < 80` 过滤

4m 处单色连通域面积 **~76 px²**，低于原 80 px² 门限，导致先验采集阶段完全检测不到球，
用户永远看不到候选列表条目，无法进入追踪。

### 瓶颈二（追踪阶段密度检测）—— `density_min_pixels = 20` 过高

4m 处单色峰值格子（grid_size=16）像素约 23 px，只比 20 略高，HSV 覆盖率稍差时
（球体侧面/阴影）即跌破门限 → 追踪频繁丢失。

### 瓶颈三（追踪阶段）—— `grid_size = 16` 过粗

4m 处球直径仅 18 px，grid_size=16 导致球最多跨 1~2 格，密度峰值不明显，
定位抖动大，进一步加剧丢失。

### 非瓶颈（无需改动）

| 参数 | 评估结论 |
|------|---------|
| `depth_diff > 0.5m` | 4m 内深度噪声 <300mm，不误拒 |
| `radius_tolerance = 0.7` | ±70% 容差已足够 |
| 形态学腐蚀 2 次 | 有膨胀补偿，净影响有限 |
| `prior_gate_px = 80` | 追踪门限合理，不改 |

## 二、改动内容（3 处参数）

### 改动 1 — `area < 80` → `area < 40`

**位置：** `_collect_candidates` 方法，第 208 行

```python
# 修改前
if area < 80:   # 过滤噪声小块（缩放后坐标系）

# 修改后
if area < 40:   # 原80在4m处会误拒单色面积约76px²，降至40保留
```

**安全性：** 先验采集结果经 `_aggregate_and_print` 距离合并去噪，再由用户人工确认目标，假阳性不会进入追踪。

---

### 改动 2 — `density_min_pixels = 20` → `10`

**位置：** `__init__`，第 78 行

```python
# 修改前
self.density_min_pixels = 20   # 峰值格子最低有效像素数，低于此视为无球

# 修改后
self.density_min_pixels = 10   # 降低灵敏度，4m 处约23px，降至10覆盖 HSV 覆盖率波动
```

**安全性：** 追踪阶段有先验位置+深度双门限二次过滤，低阈值假阳性会被先验门限拦截。

---

### 改动 3 — `grid_size = 16` → `8`

**位置：** `__init__`，第 76 行

```python
# 修改前
self.grid_size = 16   # 格子边长（像素，基于 img_scale=0.5 后分辨率）

# 修改后
self.grid_size = 8    # 细化格子，4m 处球直径18px可跨2格，定位更稳定
```

**各距离格子分析（改后）：**

| 距离 | 球直径 | 跨越格子数 | 峰值格子像素估算 |
|------|--------|----------|----------------|
| 1 m  | 72 px  | ~9×9     | ~87 px ✅       |
| 2 m  | 36 px  | ~5×5     | ~53 px ✅       |
| 3 m  | 24 px  | ~3×3     | ~47 px ✅       |
| 4 m  | 18 px  | ~2×2     | ~23 px ✅       |

**性能影响：** 缩放后图像 320×240，格子总数 40×30=1200，`histogram2d` 仍为轻量操作，帧率无显著影响。

## 四、验证方法

1. 启动节点，在 **3~4m** 处放置黄蓝球体
2. **先验采集阶段**：确认候选列表出现球体条目（验证 area 门限修复）
3. 选择目标后进入追踪：确认帧内持续检测到球（验证 density_min_pixels 修复）
4. 轻微移动球体：观察追踪稳定性（验证 grid_size 改善效果）
5. 在 **1m** 处重复验证，确认近距离识别无退化
